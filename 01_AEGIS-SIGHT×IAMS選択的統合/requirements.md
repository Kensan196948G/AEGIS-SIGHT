## 1. 目的・背景

### 1.1 統合背景

Project_Repository_List_v2_1 の分析に基づき、IntegratedITAssetServiceManagement（IAMS）が持つ有用機能を AEGIS-SIGHT に選択的に移植統合する。

AEGIS-SIGHT は SKYSEA Client View 同等機能の内製代替として開発中（v1.1）であり、IAMS の以下機能を吸収することで、エンドポイント管理とインフラ観測性・調達管理を一元化する。

移植完了後、IAMS リポジトリ（Kensan196948G/IntegratedITAssetServiceManagement）は廃止・削除する。

### 1.2 IAMS 選択移植機能の判断根拠

| 移植対象機能 | 判断理由 |
|------------|---------|
| Prometheus/Grafana インフラ可観測性 | AEGIS-SIGHT の監視機能を強化し、サーバ・インフラ層の可視化を追加 |
| PWA 対応（オフラインUI） | 建設現場での AEGIS-SIGHT 活用に直接寄与する機能 |
| 調達管理（Procurement） | IT資産ライフサイクル（購入〜廃棄）の一貫管理を実現 |
| SAMライセンス管理 | AEGIS-SIGHT の SW インベントリ機能と統合し、ライセンス超過を自動検知 |
| 1,157テスト資産 | Jest テストを pytest に変換し、AEGIS-SIGHT の品質保証に活用 |

### 1.3 IAMS から移植しない機能（廃棄理由）

| 廃棄機能 | 廃棄理由 |
|---------|---------|
| CMDB | AEGIS-SIGHT 本体の資産管理 DB と完全重複 |
| インシデント管理 | ITSM-System と完全重複 |
| SLA 管理 | ITSM-System と完全重複 |
| 変更管理 | ITSM-System（統合プランA）と重複 |

## 3. スコープ

### 3.1 IAMS から移植する機能（選択的移植）

#### 機能①：Prometheus/Grafana インフラ可観測性（移植）

| 要件ID | 要件名 | 優先度 | 詳細 |
|--------|--------|--------|------|
| OBS-001 | Prometheus メトリクス収集 | 必須 | Windows エージェントからサーバリソース（CPU/メモリ/ディスク/ネットワーク）のメトリクスを Prometheus 形式で収集 |
| OBS-002 | Grafana 統合ダッシュボード | 必須 | インフラ状態・エンドポイント状態を Grafana でリアルタイム可視化 |
| OBS-003 | アラートルール設定 | 必須 | CPU 90%超過・ディスク残量 10%未満・サービス停止等のアラートを設定 |
| OBS-004 | AEGIS-SIGHT ダッシュボードへの埋め込み | 推奨 | Grafana パネルを AEGIS-SIGHT の Next.js ダッシュボードに iframe 埋め込み |

#### 機能②：PWA 対応（オフライン UI）（移植）

| 要件ID | 要件名 | 優先度 | 詳細 |
|--------|--------|--------|------|
| PWA-001 | Service Worker 実装 | 必須 | ネットワーク断絶時にキャッシュされた資産情報を参照可能 |
| PWA-002 | オフライン資産確認機能 | 必須 | 建設現場でネットワーク不通時でも端末の割当・状態を確認できる |
| PWA-003 | オンライン復帰時の自動同期 | 必須 | ネットワーク復帰後にオフライン操作（状態更新等）を自動同期 |
| PWA-004 | ホーム画面追加対応 | 推奨 | スマートフォン・タブレットのホーム画面に追加可能なマニフェスト設定 |

#### 機能③：調達管理（Procurement）（移植）

| 要件ID | 要件名 | 優先度 | 詳細 |
|--------|--------|--------|------|
| PRC-001 | 調達申請・承認ワークフロー | 必須 | IT機器・ソフトウェアの調達申請〜上長承認〜発注のワークフロー |
| PRC-002 | 受領登録・資産台帳連携 | 必須 | 機器受領時に自動で AEGIS-SIGHT 資産台帳に登録 |
| PRC-003 | 廃棄管理 | 必須 | 廃棄申請・廃棄証明書管理・台帳からの除却記録 |
| PRC-004 | 調達ライフサイクル追跡 | 必須 | 調達→受領→使用中→保守→廃棄の全ライフサイクルを一元管理 |
| PRC-005 | 費用管理 | 推奨 | 調達費用・保守費用の記録・部門別コスト集計 |

#### 機能④：SAM ライセンス管理（移植）

| 要件ID | 要件名 | 優先度 | 詳細 |
|--------|--------|--------|------|
| SAM-001 | ライセンス登録・管理 | 必須 | ソフトウェアライセンス（本数・有効期限・ライセンス種別）の登録・管理 |
| SAM-002 | インベントリとの自動照合 | 必須 | AEGIS-SIGHT の SW インベントリ収集結果とライセンス登録数を自動照合 |
| SAM-003 | ライセンス超過アラート | 必須 | インストール数がライセンス購入数を超過した場合に自動アラート |
| SAM-004 | ライセンス期限アラート | 必須 | 有効期限 30日前・7日前のアラート通知 |
| SAM-005 | SAM レポート（J-SOX） | 必須 | IT 全般統制対応のライセンス管理証跡レポートを自動生成 |
| SAM-006 | Microsoft 365 ライセンス連携 | 推奨 | Microsoft Graph API 経由で M365 ライセンス割当状況を自動取得 |

#### 機能⑤：テスト資産の pytest 変換活用

| 要件ID | 要件名 | 優先度 | 詳細 |
|--------|--------|--------|------|
| TST-001 | IAMS テスト資産分析 | 必須 | IAMS の 1,157件の Jest テストを分析し、AEGIS-SIGHT に移植すべきテストケースを選定 |
| TST-002 | pytest 変換・統合 | 必須 | 選定したテストケースを pytest 形式に変換し、AEGIS-SIGHT のテストスイートに統合 |
| TST-003 | カバレッジ目標達成 | 必須 | テスト変換により AEGIS-SIGHT のカバレッジを 80%以上に引き上げ |

## 4. 既存 AEGIS-SIGHT 機能との統合要件

本要件は、既存の AEGIS-SIGHT_要件定義書_v1.1.docx に記載の機能と統合される。

| AEGIS-SIGHT 既存機能 | IAMS 移植機能 | 統合方針 |
|-------------------|-------------|---------|
| IT資産管理（HW情報収集） | 調達管理 | 調達→台帳自動登録のライフサイクル統合 |
| SW インベントリ管理 | SAM ライセンス管理 | インベントリ収集結果でライセンス照合を自動実行 |
| 統合ダッシュボード（Next.js） | Grafana インフラ監視 | ダッシュボードへの Grafana パネル埋め込み |
| 現場PC（オフラインバッファ対応） | PWA オフライン UI | 既存オフラインバッファ設計に PWA レイヤを追加 |

## 5. 非機能要件

| 要件区分 | 要件 | 基準値 |
|---------|------|--------|
| **可用性** | サービス稼働率 | 99.5%以上（AEGIS-SIGHT 本体と同一） |
| **パフォーマンス** | ライセンス照合処理 | 500件のSWインベントリを5秒以内に照合 |
| **オフライン対応** | PWA キャッシュサイズ | 最大 50MB（直近30日分の資産情報） |
| **スケーラビリティ** | 管理資産数 | 端末 500台・ライセンス 1,000件以上に対応 |
| **セキュリティ** | 調達データ保護 | 費用情報は管理者ロールのみアクセス可 |
| **テスト** | カバレッジ | 統合後 80%以上（AEGIS-SIGHT 本体と合算） |
| **ログ保存** | 調達・廃棄記録 | 7年保存（J-SOX 対応） |

## 7. 受入れ条件

- 全 IAMS 移植機能（OBS/PWA/PRC/SAM/TST）の実装・単体テスト合格
- ライセンス超過検知が正常動作すること（テストデータで検証）
- Prometheus/Grafana ダッシュボードが稼働中の端末データを表示すること
- PWA がオフライン時に直近の資産情報を表示できること
- AEGIS-SIGHT テストカバレッジが 80%以上であること
- IAMS リポジトリの廃止承認
