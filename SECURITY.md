# Security Policy

## Supported Versions

現在セキュリティアップデートを提供しているバージョンは以下のとおりです。

| Version | Supported          |
|---------|--------------------|
| 0.24.x  | :white_check_mark: |
| 0.23.x  | :white_check_mark: |
| < 0.23  | :x:                |

## Reporting a Vulnerability

セキュリティ上の脆弱性を発見した場合は、以下の手順で報告をお願いします。

### 報告方法

1. **GitHub Security Advisories** (推奨)
   - [Security Advisories](https://github.com/Kensan196948G/AEGIS-SIGHT/security/advisories/new) から非公開の報告を作成してください。

2. **メール**
   - セキュリティチーム宛にメールを送信してください。
   - 件名に `[SECURITY] AEGIS-SIGHT` を含めてください。

### 報告に含めるべき情報

- 脆弱性の種類 (XSS, SQLi, SSRF, etc.)
- 影響を受けるコンポーネント (API, Web, Agent, Infra)
- 再現手順 (ステップバイステップ)
- 影響範囲の評価
- 可能であれば、修正案

### 対応プロセス

1. **確認 (48時間以内):** 報告の受領確認を返信します。
2. **トリアージ (5営業日以内):** 脆弱性の深刻度を評価します。
3. **修正 (深刻度に応じて):**
   - **Critical/High:** 7日以内にパッチリリース
   - **Medium:** 次回リリースに含める
   - **Low:** バックログに追加
4. **通知:** 修正リリース後、報告者に通知します。

### 深刻度の分類

| 深刻度 | CVSS Score | 対応期限 |
|--------|------------|----------|
| Critical | 9.0 - 10.0 | 72時間 |
| High | 7.0 - 8.9 | 7日 |
| Medium | 4.0 - 6.9 | 30日 |
| Low | 0.1 - 3.9 | 次回リリース |

## Security Best Practices

AEGIS-SIGHT を安全に運用するためのガイドラインです。

### 必須設定

- `SECRET_KEY` を本番環境では必ず変更してください (最低32文字のランダム文字列)
- `DEBUG=False` を本番環境で設定してください
- HTTPS を必ず使用してください
- データベースのデフォルトパスワードを変更してください

### 推奨設定

- IP アドレス制限を設定してください (`ALLOWED_IPS`)
- レート制限を有効にしてください
- ログを外部の SIEM に転送してください
- 定期的にセキュリティスキャンを実行してください (`Trivy`)

### アクセス制御

- 最小権限の原則に従ってください
- RBAC ロールを適切に設定してください
- 管理者アカウントには MFA を有効にしてください
- 不要なアカウントは速やかに無効化してください

## Disclosure Policy

- 脆弱性の公開は修正リリース後に行います。
- 報告者のクレジットを公開時に記載します (希望する場合)。
- Public Issue での脆弱性の報告は行わないでください。

## Dependencies

依存パッケージのセキュリティは以下で管理しています。

- **Dependabot:** 自動的に依存パッケージのアップデート PR を作成
- **Trivy:** Docker イメージのセキュリティスキャン (CI/CD)
- **Ruff:** Python コードの静的解析
