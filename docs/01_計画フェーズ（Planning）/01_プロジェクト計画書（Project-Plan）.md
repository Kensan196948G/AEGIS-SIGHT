## 1. プロジェクト概要

### 1.1 プロジェクト名称

**AEGIS-SIGHT（Autonomous Endpoint Governance & Integrated Sight）**
--- IAMS 選択的統合プロジェクト

### 1.2 プロジェクトの背景

みらい建設工業（従業員約550名、管理対象端末約500台）では、SKYSEA Client View同等のITエンドポイント管理機能を内製化する「AEGIS-SIGHT」プロジェクトを推進している。

既存の IntegratedITAssetServiceManagement（IAMS）リポジトリに蓄積された有用機能（Prometheus/Grafana、PWA、調達管理、SAMライセンス管理、テスト資産1,157件）を AEGIS-SIGHT へ選択的に移植統合することで、エンドポイント管理とインフラ観測性・調達管理を一元化する。

### 1.3 プロジェクトの目的

| # | 目的 | 達成指標（KPI） |
|---|------|----------------|
| 1 | 商用製品依存からの脱却 | SKYSEA Client View ライセンスコスト 70%以上削減 |
| 2 | IT資産ライフサイクルの一元管理 | 調達～廃棄まで単一システムで管理率 100% |
| 3 | インフラ可観測性の統合 | サーバ・端末メトリクスの Grafana 一元可視化 |
| 4 | 建設現場でのオフライン運用 | PWA対応によるオフライン閲覧可能率 100% |
| 5 | ライセンスコンプライアンスの自動化 | ライセンス超過の自動検知率 100% |
| 6 | コンプライアンス準拠 | ISO27001, ISO20000-1, NIST CSF 2.0, J-SOX 全項目準拠 |

### 1.4 プロジェクト憲章（チャーター）

## 2. スコープ定義

### 2.1 スコープ内（In-Scope）

#### 2.1.1 IAMS から移植する機能

| 機能カテゴリ | 移植内容 | 優先度 |
|------------|---------|--------|
| Prometheus/Grafana インフラ可観測性 | メトリクス収集、統合ダッシュボード、アラートルール、Next.js埋め込み | 必須 |
| PWA 対応（オフライン UI） | Service Worker、オフライン資産確認、自動同期、ホーム画面追加 | 必須 |
| 調達管理（Procurement） | 申請/承認ワークフロー、受領登録、廃棄管理、ライフサイクル追跡、費用管理 | 必須 |
| SAM ライセンス管理 | ライセンス登録/管理、インベントリ自動照合、超過アラート、J-SOXレポート | 必須 |
| テスト資産変換 | Jest 1,157件 → pytest への変換、品質保証フレームワーク構築 | 必須 |

#### 2.1.2 基盤作業

- データベーススキーマ設計・マイグレーション（PostgreSQL 16）
- API設計・実装（Python FastAPI）
- フロントエンド統合（Next.js 14）
- Docker コンテナ構成の統合
- PowerShell Agent のメトリクス収集拡張
- CI/CD パイプライン構築

### 2.2 スコープ外（Out-of-Scope）

| 除外項目 | 理由 |
|---------|------|
| CMDB | AEGIS-SIGHT本体の資産管理DBと完全重複 |
| インシデント管理 | ITSM-Systemと完全重複 |
| SLA管理 | ITSM-Systemと完全重複 |
| 変更管理 | ITSM-System（統合プランA）と重複 |
| ハードウェア調達（物理購買プロセス） | 既存の社内購買システムを利用 |
| 外部システムとの新規API連携開発 | フェーズ2以降で検討 |

## 7. 前提条件

| # | 前提条件 | リスク（前提崩壊時） |
|---|---------|-------------------|
| 1 | IT部門メンバー5名が本プロジェクトに参画可能 | リソース不足による遅延 |
| 2 | 既存 IAMS リポジトリのソースコードが完全に利用可能 | 移植作業の手戻り |
| 3 | Docker / PostgreSQL 16 の本番環境が利用可能 | インフラ構築遅延 |
| 4 | ClaudeCode による自律型開発が計画通りの生産性を発揮 | 開発期間延長 |
| 5 | 社内ネットワークで Prometheus / Grafana の通信が許可される | 監視基盤構築不可 |
| 6 | Windows 11/10 端末への PowerShell Agent 展開が可能 | エージェント配布遅延 |
| 7 | 経営層からのプロジェクト承認が W2 までに完了 | プロジェクト開始遅延 |

## 8. 制約条件

| # | 制約条件 | 対応方針 |
|---|---------|---------|
| 1 | 開発期間は20週間（延長不可） | スコープ調整で対応、優先度による段階リリース |
