## 1. 目的

本文書は、AEGIS-SIGHT IAMS選択的統合プロジェクトに関わる全ステークホルダーを特定し、各ステークホルダーの関心事項・影響力・関与レベルを分析した上で、適切なエンゲージメント戦略を定義するものである。

## 5. エンゲージメント戦略

### 5.1 密接に管理（Manage Closely）

#### SH-001: 経営企画部長（スポンサー）

| 項目 | 内容 |
|------|------|
| **エンゲージメント目標** | プロジェクトへの強い支持を維持し、経営レベルの障壁除去に協力を得る |
| **コミュニケーション方法** | 月次経営報告（対面）、重要マイルストーン時の個別報告 |
| **頻度** | 月1回 + マイルストーン時（随時） |
| **キーメッセージ** | ROI進捗、コスト削減実績、リスク状況、経営判断が必要な事項 |
| **担当** | PM |
| **アクション** | Phase 2 完了時にコスト削減効果の中間報告を実施 |

#### SH-002: IT部門長（承認者）

| 項目 | 内容 |
|------|------|
| **エンゲージメント目標** | 技術的意思決定の迅速な承認、リソース確保への支援 |
| **コミュニケーション方法** | 週次報告（対面/オンライン）、承認依頼時の個別ミーティング |
| **頻度** | 週1回 + 承認時（随時） |
| **キーメッセージ** | 進捗状況、品質指標、リスク・課題、承認依頼事項 |
| **担当** | PM |
| **アクション** | 週次定例で承認事項を事前にアジェンダ化し、迅速な意思決定を促進 |

#### SH-015: 内部監査室

| 項目 | 内容 |
|------|------|
| **エンゲージメント目標** | 監査要件の早期確認と、設計段階からの監査適合性の確保 |
| **コミュニケーション方法** | フェーズ開始時のレビュー会議、証跡設計の事前協議 |
| **頻度** | 各フェーズ開始/終了時 |
| **キーメッセージ** | 監査証跡設計、J-SOX対応状況、コンプライアンスチェック結果 |
| **担当** | PM |
| **アクション** | Phase 1 で監査室との事前協議を実施し、証跡要件を確定 |

### 5.2 情報提供を維持（Keep Informed）

#### SH-004 ~ SH-008: 開発チーム

| 項目 | 内容 |
|------|------|
| **エンゲージメント目標** | 技術的モチベーション維持、チーム間の情報共有促進 |
| **コミュニケーション方法** | デイリースタンドアップ、Slack チャンネル、Sprint レビュー |
| **頻度** | 毎日（スタンドアップ）、随時（Slack） |
| **キーメッセージ** | タスク進捗、技術的課題、設計変更、ベストプラクティス |
| **担当** | TL |
| **アクション** | Sprint ごとのレトロスペクティブでフィードバック収集 |

#### SH-009: IT運用チーム

| 項目 | 内容 |
|------|------|
| **エンゲージメント目標** | 運用視点のフィードバック取得、スムーズな運用引き継ぎ |
| **コミュニケーション方法** | Phase 3 からの運用レビュー参加、運用手順書の共同作成 |
| **頻度** | Phase 3 以降月2回 |
| **キーメッセージ** | 運用影響、監視アラート設計、障害対応フロー |
| **担当** | PM/TL |
| **アクション** | Phase 3 開始時に運用チームをレビューアーに追加 |

#### SH-011 ~ SH-014: 利用部門

| 項目 | 内容 |
|------|------|
| **エンゲージメント目標** | 移行への理解と協力、現場要件の吸い上げ |
| **コミュニケーション方法** | 月次ニュースレター、Phase 3 でのデモセッション |
| **頻度** | 月1回 |
| **キーメッセージ** | プロジェクト進捗、業務影響、移行スケジュール、新機能紹介 |
| **担当** | PM |
| **アクション** | Phase 3 完了時にデモセッションを全部門管理者向けに開催 |

### 5.3 満足を維持（Keep Satisfied）

#### SH-017: 外部監査法人

| 項目 | 内容 |
|------|------|
| **エンゲージメント目標** | J-SOX監査への円滑な対応、証跡の事前確認 |
| **コミュニケーション方法** | フェーズ完了時の報告書共有 |
| **頻度** | 四半期1回 |
| **キーメッセージ** | 内部統制設計の変更点、証跡管理体制 |
| **担当** | PM/コンプライアンス担当 |
