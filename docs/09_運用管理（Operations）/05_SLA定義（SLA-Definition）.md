## 1. SLA概要

AEGIS-SIGHTの内部SLA（Service Level Agreement）を定義する。本SLAは社内向けサービスとしての品質目標であり、運用チームと利用部門間の合意に基づく。

---

## 2. 可用性

### 2.1 可用性目標

| メトリクス | 目標値 | 計算方法 |
|-----------|--------|---------|
| 月間可用性 | **99.5%** | (稼働時間 / 計測時間) x 100 |
| 年間可用性 | **99.5%** | (年間稼働時間 / 年間計測時間) x 100 |

### 2.2 許容ダウンタイム

| 期間 | 可用性99.5%での許容ダウンタイム |
|------|------------------------------|
| 月間 | 約3時間36分 |
| 四半期 | 約10時間57分 |
| 年間 | 約43時間48分 |

### 2.3 計測除外事項

以下の時間帯・事象は可用性計算から除外する。

| 除外事項 | 説明 |
|---------|------|
| 計画保守窓 | 毎月第2日曜 2:00-6:00（事前通知済み） |
| 緊急保守 | CTO承認済み、事前通知済みの緊急メンテナンス |
| 不可抗力 | 自然災害、停電、ネットワーク事業者障害 |
| 利用部門起因 | 利用者側のネットワーク障害、端末障害 |

### 2.4 可用性レポート

| 項目 | 頻度 | 報告先 |
|------|------|--------|
| 月次可用性レポート | 月次（翌月第1週） | 情報システム部長、利用部門代表 |
| 四半期レビュー | 四半期 | CTO、利用部門 |
| 年次SLAレビュー | 年次 | 経営層 |

## 4. 障害対応時間

### 4.1 対応時間目標

| 障害レベル | 検知時間 | 対応開始 | 復旧目標 |
|-----------|---------|---------|---------|
| P1 (Critical) | < 5分（自動監視） | < 15分 | < 1時間 |
| P2 (Major) | < 15分 | < 30分 | < 4時間 |
| P3 (Minor) | < 1時間 | 翌営業日 | 3営業日以内 |
| P4 (Low) | 翌営業日 | 次スプリント | 次リリース |

### 4.2 対応時間帯

| 区分 | 時間帯 | P1対応 | P2対応 | P3/P4対応 |
|------|--------|--------|--------|----------|
| 営業時間 | 平日 9:00-18:00 | 対応 | 対応 | 対応 |
| 準備対応 | 平日 18:00-22:00 | 対応 | 対応 | 翌営業日 |
| 時間外 | 22:00-9:00、土日祝 | 対応 | 翌営業日 | 翌営業日 |

## 5. データ保護

### 5.1 バックアップ・復旧目標

| メトリクス | 目標値 | 説明 |
|-----------|--------|------|
| RPO（目標復旧地点） | 1時間 | 最大1時間分のデータ損失を許容 |
| RTO（目標復旧時間） | 1時間 | 障害発生から1時間以内に復旧 |
| バックアップ頻度 | 日次 + WAL連続 | 日次フルバックアップ + WALアーカイブ |
| バックアップ保持期間 | 30日 | 30日分のバックアップを保持 |
| 復旧テスト | 月次 | 毎月バックアップからの復旧テストを実施 |

### 5.2 データ整合性

| 項目 | 目標 |
|------|------|
| データ整合性チェック | 週次で自動実行 |
| トランザクション完全性 | ACID準拠（PostgreSQL保証） |
| 監査ログ保持 | 1年間 |

## 9. SLAメトリクスダッシュボード

### 9.1 主要KPI

| KPI | 計算式 | 目標 | 報告頻度 |
|-----|--------|------|---------|
| 可用性 | 稼働時間 / 計測時間 x 100 | 99.5% | 月次 |
| MTTR | 障害復旧時間の平均 | < 1時間（P1） | 月次 |
| MTBF | 障害間隔の平均 | > 720時間（30日） | 四半期 |
| P95応答時間 | 95パーセンタイル | < 500ms | 週次 |
| エラー率 | 5xxレスポンス / 全リクエスト | < 0.5% | 日次 |
| バックアップ成功率 | 成功回数 / 実行回数 | 100% | 月次 |
| SLAブリーチ件数 | SLA違反の件数 | 0件 | 月次 |

### 9.2 月次SLAレポートテンプレート

```
AEGIS-SIGHT SLAレポート - YYYY年MM月

■ 可用性
  月間可用性: XX.XX%（目標: 99.5%）
  計画外ダウンタイム: XX分
  計画保守: XX分（除外）

■ 応答時間
  API p50: XXXms（目標: < 200ms）
  API p95: XXXms（目標: < 500ms）
  API p99: XXXXms（目標: < 1000ms）

■ 障害
  P1障害: X件（MTTR: XX分）
  P2障害: X件（MTTR: XX分）
  P3障害: X件
  P4障害: X件

■ バックアップ
  日次バックアップ: XX/XX 成功（成功率: XXX%）
  復旧テスト: 実施 / 未実施

■ セキュリティ
  脆弱性スキャン: 実施（Critical: X, High: X）
  セキュリティパッチ: X件適用

■ SLAブリーチ
  件数: X件
  詳細: <ブリーチがあった場合の説明>

■ 総合評価
  SLA達成: 達成 / 未達成
  特記事項: <特記事項があれば記載>
```

## 10. SLAレビューと改善

### 10.1 レビュースケジュール

| レビュー | 頻度 | 参加者 | 議題 |
|---------|------|--------|------|
| 月次レビュー | 毎月 | 運用チーム | SLAメトリクス確認、改善アクション |
| 四半期レビュー | 四半期 | CTO + 運用チーム + 利用部門 | SLA目標の妥当性検証 |
| 年次レビュー | 年次 | 経営層 + 全関係者 | SLA目標の見直し |

### 10.2 SLA改善プロセス

```
SLAブリーチ検出
    │
    ▼
根本原因分析
    │
    ▼
改善策策定
    │
    ▼
改善実施
    │
    ▼
効果測定
    │
    ▼
SLA目標見直し（必要時）
```
