## 1. 本文書の目的

AEGIS-SIGHT への IAMS 選択的統合プロジェクトにおける受入れ条件を、フェーズ別および最終受入れの観点から定義する。本文書は受入試験（UAT）の合否判定基準として使用する。

---

## 2. 受入れ条件の判定基準

### 2.1 判定区分

| 判定 | 定義 | 対応 |
|------|------|------|
| **合格（Pass）** | 受入条件を全て満たしている | 次フェーズに進行 / リリース承認 |
| **条件付き合格（Conditional Pass）** | 必須条件は満たすが、推奨条件に未達がある | 是正計画を提出の上で次フェーズに進行。未達項目は次フェーズで解消。 |
| **不合格（Fail）** | 必須条件に未達がある | 再実装・再テスト後に再審査 |

### 2.2 品質レベル定義

| レベル | 名称 | 定義 |
|--------|------|------|
| L1 | 基本動作 | 正常系のシナリオが動作すること |
| L2 | 異常系対応 | エラー処理・バリデーション・境界値が適切に処理されること |
| L3 | 性能・負荷 | 非機能要件の性能基準を満たすこと |
| L4 | セキュリティ | セキュリティ要件を満たし、脆弱性スキャンに合格すること |
| L5 | 運用適性 | ログ・監視・バックアップが正常に機能し、運用手順が整備されていること |

| プレ運用期間 | 2 週間 | 本番環境での安定稼働確認 |

## 8. 不合格時の対応手順

1. **不合格項目の特定**: 不合格となった受入条件と具体的な不適合内容を記録する
2. **原因分析**: 不適合の根本原因を分析し、是正方針を策定する
3. **是正計画書の作成**: 是正内容・担当者・完了予定日を記載した是正計画書を作成する
4. **是正実施**: 開発チームが是正を実施する
5. **再テスト**: 是正完了後に該当する受入条件の再テストを実施する
6. **再審査**: 再テスト結果を基に受入れの再審査を行う

再審査は最大 2 回まで。3 回目の不合格でプロジェクトのエスカレーション（IT 部門長 + 経営層への報告）を行う。
