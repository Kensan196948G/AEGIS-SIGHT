## 2. IAMS移植対象の整理

### 移植する機能

| 機能 | 移植理由 |
|------|----------|
| Prometheus/Grafana監視基盤 | AEGIS-SIGHTのエンドポイント監視に必須 |
| PWA（Progressive Web App） | 現場端末でのオフライン利用要件 |
| 調達管理 | IT資産の調達ワークフローをAEGIS-SIGHTに統合 |
| SAMライセンス管理 | ソフトウェアライセンスのコンプライアンス管理 |
| テスト資産（1,157件） | Jest→pytestへ変換し品質基盤を確立 |

### 移植しない機能

| 機能 | 除外理由 |
|------|----------|
| CMDB（構成管理データベース） | ITSM-Systemと重複 |
| インシデント管理 | ITSM-Systemと重複 |
| SLA管理 | ITSM-Systemと重複 |
| 変更管理 | ITSM-Systemと重複 |

## 4. クリティカルパス

```
Phase1(2w) → Phase2(6w) → Phase4(4w) = 合計12週間（約3ヶ月）
Phase1(2w) → Phase3(6w) → Phase4(4w) = 合計12週間（約3ヶ月）
```

Phase2とPhase3は並行実施可能であり、どちらも同じ長さのパスを持つ。
クリティカルパスは以下の2本が同時に存在する。

1. **Phase1 → Phase2 → Phase4**: SAM・調達の実装完了がPhase4のテストに必須
2. **Phase1 → Phase3 → Phase4**: 監視・PWAの実装完了がPhase4のテストに必須

### リスク軽減策

- Phase2/Phase3の並行開発体制を確保し、どちらかの遅延がPhase4に波及しないようバッファを設ける
- Phase4のテスト変換（Jest→pytest）はPhase2/Phase3の完了を待たずにフレームワーク準備を先行実施
- 各フェーズ終了時にGo/No-Goレビューを実施
