# 01. システムアーキテクチャ全体図（System Architecture）

## 1. 概要

AEGIS-SIGHTは、IT資産管理・ソフトウェア資産管理（SAM）・調達管理を統合的に行うWebベースのプラットフォームである。オンプレミス環境およびクラウド環境の両方に対応し、オフライン現場でもPWAを通じた運用が可能な設計としている。

## 3. レイヤー構成

| レイヤー | 技術スタック | 責務 |
|---------|-------------|------|
| クライアント層 | PowerShell Agent, PWA | 資産情報収集、ユーザーインターフェース |
| プレゼンテーション層 | Next.js 14 + TypeScript + Tailwind | SPA/PWA、SSR、UIレンダリング |
| API層 | FastAPI (Python) | RESTful API、認証・認可、ビジネスロジック |
| データ層 | PostgreSQL 16, Redis | 永続化、キャッシュ、セッション管理 |
| インフラ層 | Docker Compose, Nginx | コンテナオーケストレーション、リバースプロキシ |
| 外部連携層 | Microsoft Graph, Prometheus | M365統合、監視・メトリクス収集 |

## 4. 通信プロトコル

| 通信経路 | プロトコル | 認証方式 |
|---------|----------|---------|
| クライアント - Nginx | HTTPS (TLS 1.3) | JWT + OIDC |
| Nginx - Next.js | HTTP (内部通信) | - |
| Next.js - FastAPI | HTTP (内部通信) | JWT Bearer |
| FastAPI - PostgreSQL | TCP/5432 | ユーザー認証 + SSL |
| FastAPI - Redis | TCP/6379 | パスワード認証 |
| FastAPI - Microsoft Graph | HTTPS | OAuth 2.0 |
| Windows Agent - FastAPI | HTTPS | mTLS + API Key |
| Prometheus - FastAPI | HTTP | Basic認証 |

## 5. データフロー

### 5.1 資産情報収集フロー

```
Windows PC
  │
  ▼
PowerShell Agent (WMI/Prometheus Exporter)
  │
  ▼ HTTPS + mTLS
FastAPI (/api/v1/assets)
  │
  ├──▶ PostgreSQL (資産DB)
  └──▶ Redis (キャッシュ更新)
```

### 5.2 オフライン現場フロー

```
現場PC (オフライン)
  │
  ▼
PWA (Service Worker + IndexedDB バッファ)
  │
  ▼ オンライン復帰時に自動同期
FastAPI (/api/v1/assets)
  │
  ▼
PostgreSQL (差分マージ)
```

### 5.3 SAM・調達フロー

```
管理者 (Web UI)
  │
  ▼
Next.js (PWA)
  │
  ▼
FastAPI (/api/v1/sam, /api/v1/procurement)
  │
  ├──▶ PostgreSQL (SAM DB, 調達DB)
  └──▶ Microsoft Graph API (M365ライセンス情報取得)
```

## 6. 可用性・スケーラビリティ

### 6.1 水平スケーリング

- FastAPIコンテナは複数レプリカで水平スケーリング可能
- Nginx上流でラウンドロビンによる負荷分散
- Redisによるセッション共有で状態管理

### 6.2 障害対策

| 対策 | 内容 |
|------|------|
| DBバックアップ | PostgreSQL pg_dump による日次バックアップ |
| オフラインバッファ | PWA Service Workerによるオフラインデータ保持 |
| ヘルスチェック | Docker HEALTHCHECK + Prometheus監視 |
| ログ集約 | 構造化ログ（JSON）による一元管理 |
